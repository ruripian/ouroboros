# 프로젝트

[English](Projects) · **한국어**

프로젝트는 워크스페이스 내부의 조직 단위입니다. 각 프로젝트는 고유한 이슈·상태·카테고리·스프린트·라벨·이벤트·멤버를 가집니다. 한 워크스페이스는 원하는 만큼의 프로젝트를 담을 수 있습니다.

## 프로젝트 생성

사이드바의 **PROJECTS** 옆 **+** 버튼을 누르거나, 워크스페이스 홈에서 **Create project** 버튼을 사용합니다. 아래 다이얼로그가 열립니다:

![프로젝트 생성 다이얼로그](https://raw.githubusercontent.com/ruripian/OrbiTail/main/docs/wiki-screenshots/05-project-create.png)

입력 항목:

- **Project Name** — 앱 전반에 표시되는 이름.
- **Identifier** — 이슈 ID 접두어로 사용되는 짧은 대문자 코드 (예: `AUR` → `AUR-1`, `AUR-2`…). 나중에 변경 불가.
- **Project Lead** — 주요 책임자. 기본값은 본인.
- **Visibility**
  - **Private** — 명시적으로 초대한 멤버만 접근 가능.
  - **Public** — 워크스페이스 모든 멤버가 볼 수 있음. 비멤버는 **Discover** 를 통해 프로젝트를 탐색하고 원할 때 참가할 수 있습니다.
- **Description** — 선택적 맥락 설명.
- **Members** — 생성 시점에 초대할 사람들. 나중에 언제든 추가 가능.

새로 만든 프로젝트에는 기본 상태 5개 (Backlog, Todo, In Progress, Done, Cancelled) 가 자동으로 세팅되며, **Workspace settings → States** 에서 커스터마이즈할 수 있습니다.

## 프로젝트 멤버와 세분화 권한

각 프로젝트는 자체 멤버 리스트를 가지며, 워크스페이스 멤버의 부분 집합입니다. 프로젝트 내부에서 **Workspace settings → Members** 를 열어 관리합니다.

![프로젝트 멤버와 권한](https://raw.githubusercontent.com/ruripian/OrbiTail/main/docs/wiki-screenshots/05-project-members.png)

각 멤버는 **역할** 을 가집니다:

- **Admin** — 멤버 추가/제거 포함 모든 작업 가능.
- **Member** — 프로젝트에서 일할 수 있으나 아래 세분화 권한의 제약을 받음.
- **Viewer** — 읽기 전용. 보는 것만 가능.

멤버 리스트 아래에 **Granular permissions** 매트릭스가 있습니다. 비Admin 멤버별로 독립적으로 허용/차단할 수 있는 플래그:

- **Edit** — 이슈 필드 수정, 인라인 편집, 벌크 액션.
- **Archive** — 이슈나 프로젝트 자체 보관.
- **Trash** — 이슈 소프트 삭제.
- **Purge** — 휴지통에서 이슈 영구 삭제.

Admin 은 체크박스와 무관하게 모든 플래그를 자동으로 가집니다.

## 프로젝트 보관

끝난 프로젝트를 잃지 않고 치우고 싶다면 보관할 수 있습니다. 보관된 프로젝트는 메인 사이드바에서 숨겨지지만 사이드바 상단의 **Archived** 에서 여전히 접근 가능합니다.

![보관된 프로젝트](https://raw.githubusercontent.com/ruripian/OrbiTail/main/docs/wiki-screenshots/05-archived-projects.png)

각 보관 프로젝트에는 **Restore** 버튼이 있어 이슈와 이력이 온전한 상태로 활성 리스트로 복귀할 수 있습니다. 보관은 완전히 되돌릴 수 있는 소프트 삭제입니다 — 이 방식으로는 어떤 데이터도 영구 삭제되지 않습니다.
