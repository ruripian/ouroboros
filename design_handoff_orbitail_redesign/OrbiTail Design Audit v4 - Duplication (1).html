03두 설정의 겹침

같은 단어 “Notifications”가 user settings와 project settings 양쪽에 있다. 뭐가 어디서 결정되는지 사용자가 알 길이 없다. 가장 큰 인지 부조화 지점.

03-A

“Notifications”가 두 곳에 — 책임 분리 필요

naming + IA fix
User · /settings/preferences
  • Email 마스터 on/off
  • 이슈 배정 시 메일
  • 댓글 작성 시 메일
  • 이슈 업데이트 시 메일

→ “내가 어떻게 받을지”

Project · /settings/notifications
  • 프로젝트 단위 메일 on/off
  • 이슈 생성 시
  • 상태 변경 시
  • 댓글 시
  • 웹훅 / Slack 통합

→ “이 프로젝트가 무엇을 발신할지”

두 영역은 본질적으로 다르다 — 하나는 사용자 수신 설정, 하나는 프로젝트 발신/통합 설정. 그런데 같은 “Notifications” 라벨을 쓰니 사용자가 “내 알림 끄려면 어디 가야 하지?”에 답 못함.

현재 — 같은 라벨, 다른 책임
User Settings
└── Notifications     ← 내가 받는 메일 토글

Project Settings
└── Notifications     ← 같은 이름! 다른 의미
제안 — 라벨 분리
User Settings
└── Notifications     — "내가 받는 알림"
    "Email me when…"

Project Settings
└── Integrations & webhooks
    "Send updates to…"  (Slack, webhook…)

User 쪽은 “Notifications” 유지. Project 쪽은 “Integrations”로 개명 — 웹훅/Slack 같은 외부 발신 통합이 본질이고, 메일 발신은 그 한 종류로. 사용자 수신은 user 설정으로 일원화.
