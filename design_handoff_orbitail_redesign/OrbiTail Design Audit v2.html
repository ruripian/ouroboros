high

State token이 BoardView에만 살아있다

BoardView.tsx:127 TableView · backlog 등 token sprawl

BoardView는 모범적이다 — var(--state-${'{'}group{'}'}-fill)로 카운트 배지를 그린다. 그러나 TableView·BacklogView·ArchiveView·CalendarView는 여전히 state.color (사용자가 admin에서 정한 임의 hex)를 그대로 색칠한다.

결과: 같은 “In Progress” 상태가 화면마다 미묘하게 다른 채도로 보인다. 이게 “깔끔하지 않다”의 큰 원인 중 하나.

FIX · 30min
// 새 헬퍼 — lib/state-color.ts
export function stateColor(group: StateGroup, fallback?: string) {
  return {
    fill: `hsl(var(--state-${group}-fill, ${fallback ?? "220 8% 60%"}))`,
    text: `hsl(var(--state-${group}-text, ${fallback ?? "220 8% 40%"}))`,
    dot:  fallback ?? `hsl(var(--state-${group}-fill))`, // dot은 user color OK
  };
}
// 모든 view에서 import해서 일관 적용. dot(점)만 user color 허용.

04Density & rhythm

간격이 일관되지 않다. p-2.5, p-3, px-2.5 py-1, px-3 py-3… 한 화면에 4–5종의 패딩이 공존한다.

high

Board card 내부 vertical rhythm이 어긋난다

BoardView.tsx:175-241spacing

한 카드 안에서: 제목 mb-2 → 날짜 mb-2 → assignee/sequence row. 그런데 날짜가 없으면 mb-2가 통째로 사라져서 카드마다 높이가 들쭉날쭉. 칸반에서 카드 높이 차이는 시각적 노이즈다.

추가로 p-3인데 내부 row는 min-h-[20px] — 패딩이 본문 사이즈에 비해 너무 빡빡하다.

  {(start_date || due_date) && <DateRow ... />}
  <div className="flex items-center justify-between">{footer}</div>
</div>
// gap-2.5(=10px) + p-3.5(=14px) — 본문 줄높이와 비례
med

Density 토글 3단계의 폰트 비율이 어색하다

index.css:185-189type scale

현재: compact 13–15px, comfortable 14–17px, spacious 16–19px. 3단계 사이가 거의 1px 차이 — 사용자가 토글해도 “바뀐 게 맞나?” 싶을 정도다. 게다가 spacing(margin/padding)은 rem 기반이라 같이 비례 확대되지 않는 컴포넌트가 많다.

현재 polynomial — 거의 균등, 차이 미미

FIX · 5min
html[data-density="compact"]     { font-size: clamp(12px, 0.78vw, 14px); }
html[data-density="comfortable"] { font-size: clamp(14px, 0.875vw, 17px); }
html[data-density="spacious"]    { font-size: clamp(16px, 1.05vw, 20px); }
// + 카드 내부 padding도 rem화(p-3 = 0.75rem 기반 OK), 1px hardcoding 제거

BoardView.tsx:97-127

06Motion residue

Phase 1의 motion 토큰화는 깨끗했다. 하지만 코드를 grep해보면 토큰을 안 거치는 duration이 아직 18군데 남아 있다.

med

Hardcoded duration 잔재

token bypass

실제 코드에서 발견된 패턴:

7
0.22s framer transition
4
0.15s tailwind transition
5
duration-500 bg fade
12
올바르게 duration-base 사용

특히 BoardView.tsx:163에 transition={{ duration: 0.22, ease: EASE_ORBIT }}가 박혀 있는데 — 이게 --motion-base(220ms)와 같은 값이지만 토큰을 우회한다. 사용자가 reduced-motion일 때 framer-motion에 별도 지시가 안 가서 레이아웃 트윈은 여전히 발생한다.

FIX · 20min
// motion-provider.tsx에 이미 useMotion()이 있을 것 — duration 헬퍼 추가
export function useMotionDuration() {
  const { isRich, prefersReduced } = useMotion();
  return {
    fast: prefersReduced ? 0 : (isRich ? 0.12 : 0.08),
    base: prefersReduced ? 0 : (isRich ? 0.22 : 0.14),
    slow: prefersReduced ? 0 : (isRich ? 0.36 : 0.20),
  };
}
// 사용처
<motion.div transition={{ duration: m.base, ease: EASE_ORBIT }}>
low

Pulse-strip이 강조되는 색이 비슷한 두 이슈에 동시 발생하면 시각 노이즈

polish

여러 이슈가 동시에 업데이트될 때 4px stripe가 동시 점멸 — 시각적으로 “깜빡깜빡”한다. stagger를 80ms 단위로 넣으면 자연스럽게 “물결”처럼 흐른다.

med

2xs · 3xs 폰트가 11px 이하로 떨어진다

density × type

text-2xs(=10px), text-3xs(=9px)가 BoardView·dropdown에 빈번하게 등장. data-density="compact" + 노트북 12.9″에서 기본 13px × 0.75 = 9.75px — 나이가 든 사용자에게 거의 안 읽힌다.
